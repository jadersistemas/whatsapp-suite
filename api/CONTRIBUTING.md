# Contributing

Obrigado pelo interesse em contribuir com `whatsapp-go-api`.

Antes de enviar uma contribuição, leia este documento, o [LICENSE](./LICENSE), o
[NOTICE](./NOTICE), o [THIRD_PARTY_NOTICES.md](./THIRD_PARTY_NOTICES.md) e, quando
aplicável, o [CLA.md](./CLA.md).

## Licença das contribuições

Ao enviar uma contribuição, você confirma que:

- possui o direito de enviar a contribuição;
- concorda que a contribuição será disponibilizada sob `AGPL-3.0-only`;
- reconhece que o mantenedor poderá oferecer o projeto também sob licença comercial;
- não transfere automaticamente todos os seus direitos autorais, salvo mediante acordo separado;
- não enviará código incompatível, sem licença compatível ou de origem desconhecida;
- declarará código adaptado de terceiros;
- preservará notices, atribuições e licenças aplicáveis.

Contribuições externas poderão exigir aceite do [CLA.md](./CLA.md). O CLA é um
instrumento separado para permitir a manutenção de uma estratégia de
licenciamento dual, incluindo uma licença comercial futura, sem presumir que
contribuições anteriores já estejam cobertas por ele.

## Código de terceiros

Não inclua código copiado, adaptado ou gerado a partir de projetos de terceiros
sem revisar a licença aplicável. Ao enviar material derivado de terceiros,
declare claramente a origem, a licença e quaisquer alterações feitas.

Não remova avisos de copyright, notices, cabeçalhos de licença ou arquivos de
licença de terceiros.

## Escopo das contribuições

Use pull requests pequenos e focados. Explique a motivação, o comportamento
alterado e os testes executados. Para mudanças de API, endpoints, webhooks,
migrations ou contratos públicos, documente o impacto esperado.

## Segurança

Não envie tokens, chaves, arquivos `.env`, bancos de sessão, dados pessoais ou
conteúdo de mensagens reais. Se encontrar uma vulnerabilidade, não publique
detalhes exploráveis em issues públicas antes de uma triagem adequada.
